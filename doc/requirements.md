# 开源组件版本监控系统需求文档

## 1. 项目概述

`opensource-release-watcher` 是一个用于持续监控开源组件版本发布情况的 Web 服务。系统维护团队内部使用的开源组件清单，定期检查 GitHub Release 或 Tag，当发现新版本时通过邮件通知订阅人。

系统定位不是包管理器，也不是自动升级工具，而是开源组件版本变化感知服务。

## 2. 业务背景

团队项目通常依赖 protobuf、OpenCV、libevent、Eigen、zlib、OpenSSL 等开源组件。这些组件会不定期发布新版本，用于修复 Bug、修复安全漏洞、增加功能或调整接口行为。

如果完全依赖人工关注，容易出现以下问题：

- 新版本发布后无人感知。
- 安全修复没有及时同步。
- 多个项目使用不同版本，缺少统一管理。
- 组件订阅关系不明确。
- Release Note 无人跟踪。
- 升级评估和通知记录缺少留痕。

## 3. 用户角色

| 角色 | 说明 | 主要操作 |
| --- | --- | --- |
| 管理员 | 维护系统基础数据和组件清单 | 新增组件、编辑组件、配置订阅人、手动触发检查 |
| 普通订阅人 | 关注某些组件版本变化 | 接收邮件通知、查看通知记录 |

## 4. 功能范围

### 4.1 组件清单管理

系统需要支持维护开源组件基础信息：

- 组件名称。
- GitHub 仓库地址。
- 当前内部使用版本。
- 邮件订阅人。
- 检查策略。
- 是否启用检查。
- 备注信息。

### 4.2 版本检查

系统通过定时任务或手动触发执行版本检查：

- 优先查询 GitHub Release。
- 如果仓库没有 Release，则回退查询 Tag。
- 获取最新版本号、发布时间、Release Note、HTML 链接。
- 与本地记录的最近检查版本进行比较。
- 发现新版本时生成检查结果和通知任务。
- 未发现新版本时更新最近检查时间。

### 4.3 通知管理

当前阶段只支持邮件通知：

- 对不同组件配置不同订阅人。
- 新版本对同一收件人只通知一次，避免重复发送。
- 每个订阅人与组件维护独立的已通知版本基线，新订阅人只接收其基线之后的新版本通知。
- 邮件标题包含组件名称和版本变化。
- 邮件正文包含当前使用版本、最新版本、Release Note 摘要、发布时间和建议动作。
- 通知发送结果需要持久化。

### 4.4 检查结果展示

后台管理界面需要展示：

- 组件当前使用版本。
- 上游最新版本。
- 最近一次检查时间。
- 最近一次检查状态。
- 是否存在新版本。
- 是否已通知。
- Release Note 摘要。

### 4.5 通知记录查看

系统需要支持查看历史通知记录：

- 通知组件。
- 通知版本。
- 收件人。
- 通知状态。
- 失败原因。
- 发送时间。

## 5. 核心流程

```text
定时调度触发检查任务
      ↓
读取启用状态的组件清单
      ↓
查询 GitHub Release
      ↓
无 Release 时查询 Tag
      ↓
解析最新版本和 Release Note
      ↓
与本地记录版本比较
      ↓
发现新版本
      ↓
生成检查记录和邮件内容
      ↓
发送邮件给订阅人
      ↓
保存通知记录
      ↓
更新组件最近检查状态
```

## 6. 非功能需求

| 分类 | 要求 |
| --- | --- |
| 可维护性 | 前后端分离，后端按 API、service、checker、storage、notifier 分层 |
| 可靠性 | 检查失败不影响其他组件，失败原因需要记录 |
| 幂等性 | 同一组件同一版本对同一收件人不重复通知 |
| 可扩展性 | 后续可接入 GitHub Security Advisory、OSV、通知优先级和月度报告 |
| 部署复杂度 | 单机部署优先，SQLite 存储，减少外部依赖 |
| 配置方式 | Microsoft Graph 发信参数、监听端口、GitHub Token 等走服务端配置；组件数据走数据库维护 |

## 7. 版本一期边界

一期只实现以下能力：

- React + TypeScript + Vite + Ant Design 管理前端。
- Go HTTP API Server。
- SQLite 持久化。
- GitHub Release / Tag 查询。
- 定时检查和手动检查。
- 邮件通知。
- 组件、订阅人、检查记录、通知记录管理。

一期不实现以下能力：

- 自动升级依赖。
- 多租户权限系统。
- Webhook 触发。
- 非 GitHub 代码托管平台。
- 钉钉、飞书、Slack 等非邮件通知。
- 漏洞库自动扫描。
